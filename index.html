tauconf.nvim

tauconf.nvim is a customized Neovim configuration tailored to optimize your development workflow. This setup integrates essential plugins, a cohesive color scheme, and optimized keybindings to enhance productivity.

Introduction

tauconf.nvim is my personalized Neovim configuration designed to provide a comprehensive and efficient development environment. It features a collection of essential plugins, a well-chosen color scheme, and optimized keybindings, tailored for both new and experienced users.

Features

  • Plugins: Curated selection of essential plugins to enhance functionality and performance.
  • Color Scheme: `catppuccin` for a visually appealing and consistent interface.
  • Keybindings: Custom shortcuts for improved workflow and efficiency.
  • LSP Integration: Advanced Language Server Protocol support for superior code editing.
  • File Explorer: `nvim-tree` for intuitive file navigation.
  • Status Line: `lualine.nvim` for a clean and informative status bar.

Dependencies

Core Requirements

  • Neovim 0.8+: Ensure version 0.8 or later is installed.
  • Git: Required for version control and plugin management.
  • Node.js: Necessary for plugins like `telescope.nvim`.
  • Python3: Needed for Python-based plugins and LSP servers.
  • GCC/Make: Required for building some plugins (e.g., `treesitter`).
  • ripgrep: Fast search tool used by `telescope.nvim`.
  • fd: Efficient file searching tool.
  • grep: Basic search tool (typically pre-installed).

Optional Tools

  • Homebrew (macOS): For package management.
  • winget (Windows): For managing Windows packages.
  • chocolatey (Windows): Alternative package manager for Windows.

Installation

Windows

  1. Install Neovim:
    • Using winget: 
      winget install Neovim.Neovim
    • Or chocolatey: 
      choco install neovim
  2. Install Dependencies:
    • With winget: 
      winget install Git.Git Nodejs.Node Python.Python ripgrep.Grep gcc make
    • Or with chocolatey: 
      choco install git nodejs python ripgrep make gcc
  3. Update PATH: Ensure Neovim and all dependencies are included in your PATH environment variable.

macOS

  1. Install Homebrew: 
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  2. Install Neovim and Dependencies: 
    brew install neovim git node python3 ripgrep fd

Linux

  • Arch Linux: 
    sudo pacman -S neovim git nodejs python ripgrep fd gcc make
  • Debian/Ubuntu: 
    sudo add-apt-repository ppa:neovim-ppa/stable
sudo apt update
sudo apt install neovim git nodejs python3 gcc make ripgrep fd-find

    (Note: On Ubuntu, fd is fd-find. Create a symlink if necessary: ln -s $(which fdfind) ~/.local/bin/fd)

  • Fedora: 
    sudo dnf install neovim git nodejs python3 ripgrep fd-find gcc make
  • OpenSUSE: 
    sudo zypper install neovim git nodejs python3 ripgrep fd gcc make
  • Gentoo: 
    sudo emerge -av neovim git nodejs python ripgrep fd gcc make

Configuration Structure

The configuration for tauconf.nvim is organized into multiple files for better maintainability. Key files include:

  • init.lua: The main initialization file for setting up Neovim.
  • lua/plugins.lua: Manages the installation and configuration of plugins using `lazy.nvim`.
  • lua/settings.lua: Contains general settings such as indentation, line numbers, and color scheme.
  • lua/servers.lua: Configurations for LSP servers.
  • lua/backup_restore.lua: Handles backup and restore operations.
  • lua/advanced.lua: Contains advanced configurations and customizations.

Usage

After installation, open Neovim and run :PackerSync to install all plugins. Customize the configuration files as needed to match your preferences and workflow.

Advanced Configuration

For advanced users, additional configurations can be added to the lua/advanced.lua file. This may include custom keybindings, additional plugins, or advanced settings specific to your workflow.

Backup and Restore

To back up your configuration, use the backup_restore.lua script. This script allows you to create backups and restore previous configurations as needed.

Troubleshooting

If you encounter issues, check the following:

  • Dependencies: Ensure all required dependencies are installed and properly configured.
  • Configuration: Verify that your configuration files are correctly set up and free from syntax errors.
  • Plugin Installation: Run :PackerStatus to check the status of installed plugins.

FAQ

  • How do I update Neovim? Use the appropriate package manager for your system (e.g., winget on Windows, brew on macOS).
  • Can I use this configuration with other distributions? Yes, you can customize the configuration to suit different Linux distributions.
